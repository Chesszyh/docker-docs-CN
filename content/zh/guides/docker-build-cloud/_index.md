98% 的开发者每天花费长达一小时等待构建完成
（[Incredibuild: 2022 Big Dev Build Times](https://www.incredibuild.com/survey-report-2022)）。
繁重、复杂的构建可能成为开发团队的主要障碍，
同时拖慢本地开发和 CI/CD 流水线的速度。

<!-- vale Vale.Spelling = YES -->

Docker Build Cloud 通过加速镜像构建来提高开发者生产力，
减少挫败感，并帮助您缩短发布周期。

## 适用人群

- 希望解决镜像构建缓慢常见原因的用户：本地资源有限、模拟器运行缓慢以及团队间缺乏构建协作。
- 使用较旧设备但希望加快构建速度的开发者。
- 在同一代码仓库工作、希望通过共享缓存减少等待时间的开发团队。
- 执行多架构构建但不想花费数小时配置和重新构建模拟器的开发者。

## 您将学到

- 在本地和 CI 中更快地构建容器镜像
- 加速多平台镜像的构建
- 重用预构建镜像以加快工作流程

## 工具集成

与 Docker Compose、GitHub Actions 和其他 CI 解决方案配合良好

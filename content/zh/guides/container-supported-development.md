容器提供了一种一致的方式来跨不同环境构建、共享和运行应用程序。虽然容器通常用于容器化您的应用程序，但它们也使运行开发所需的基本服务变得非常容易。无需安装或连接远程数据库，您可以轻松启动自己的数据库。但可能性不止于此。

通过容器辅助开发，您可以使用容器通过模拟或运行应用程序所需服务的本地实例来增强开发环境。这提供了更快的反馈循环、更少的远程服务耦合，以及更强的错误状态测试能力。

最重要的是，无论正在开发的主应用程序是否在容器中运行，您都可以获得这些好处。

## 您将学到什么

- 容器辅助开发的含义
- 如何将非容器化应用程序连接到容器化服务
- 使用容器模拟或运行本地服务实例的多个示例
- 如何使用容器向开发环境添加额外的故障排除和调试工具

## 适用人群

- 希望减少与共享或已部署的基础设施或远程 API 端点耦合的团队
- 希望降低在开发期间直接使用云服务相关的复杂性和成本的团队
- 希望更轻松地可视化数据库、队列等内容的开发人员
- 希望在不影响应用程序本身开发的情况下简化开发环境设置的团队


## 工具集成

与 Docker Compose 和 Testcontainers 配合良好。

### 什么是容器辅助开发？

容器辅助开发是通过运行应用程序所依赖服务的本地实例或模拟器来使用容器增强开发环境的理念。一旦您开始使用容器，就可以轻松添加额外的服务来可视化或排查服务中发生的问题。

### 演示：在本地开发云应用

在开发应用程序时，将应用程序的某些方面外包给云服务（如 Amazon S3）通常更容易。但是，在本地开发中连接这些服务会引入 IAM 策略、网络限制和配置复杂性。虽然这些要求在生产环境中很重要，但它们会显著增加开发环境的复杂性。

通过容器辅助开发，您可以在开发和测试期间运行这些服务的本地实例，从而无需进行复杂的设置。在此演示中，您将看到 LocalStack 如何使完全从开发人员工作站进行应用程序开发和测试变得容易。

活动日志和洞察分析是 Docker 中用户和访问管理的有用工具。它们提供对用户操作、团队工作流程和组织趋势的可见性，有助于增强安全性、确保合规性并提高生产力。

## 活动日志

活动日志跟踪组织和仓库级别的事件，提供对仓库更改、团队更新和计费调整等活动的清晰视图。

它适用于 Docker Team 或 Docker Business 计划，数据保留三个月。

### 主要功能

 - 变更跟踪：查看更改了什么、谁进行了更改以及何时更改。

 - 全面报告：监控关键事件，如仓库创建、删除、隐私更改和角色分配。

### 示例场景

 - 安全审计跟踪：仓库的隐私设置被意外更新。活动日志显示哪个用户进行了更改以及何时更改，帮助管理员解决潜在的安全风险。

 - 团队协作审查：日志显示哪些团队成员向关键仓库推送了更新，确保在开发冲刺期间的责任明确。

 - 计费调整：跟踪谁添加或删除了订阅席位，以保持预算控制和合规性。

有关更多信息，请参阅[活动日志](/manuals/admin/organization/activity-logs.md)。

## 洞察分析

洞察分析提供 Docker 使用情况的数据驱动视图，以提高团队生产力和资源分配。

### 主要优势

 - 标准化环境：确保跨团队的一致配置并执行最佳实践。

 - 提高可见性：监控 Docker Desktop 使用情况、构建和容器活动等指标，以了解团队工作流程和参与度。

 - 优化资源：跟踪许可证使用情况和功能采用率，以最大化 Docker 订阅的价值。

### 示例场景

 - 使用趋势：识别未充分利用的许可证或资源，允许将其重新分配给更活跃的团队。

 - 构建效率：跟踪平均构建时间和成功率，以查明开发过程中的瓶颈。

 - 容器利用率：分析跨部门的容器活动，以确保适当的资源分配和成本效率。

 有关更多信息，请参阅[洞察分析](/manuals/admin/organization/insights.md)。

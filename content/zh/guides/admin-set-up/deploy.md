> [!WARNING]
> 在继续之前，请确保与用户沟通，并确认您的 IT 和 MDM 团队已准备好处理任何意外问题，因为这些步骤将影响所有登录到您的 Docker 组织的现有用户。

## 第一步：强制执行 SSO

强制执行 SSO 意味着任何拥有与您验证的域名匹配的电子邮件地址的 Docker 配置文件的用户都必须使用您的 SSO 连接登录。确保与您的 SSO 连接关联的身份提供商组涵盖您希望访问 Docker 订阅的所有开发人员组。

## 第二步：向用户部署配置设置并强制登录

让 MDM 团队将 Docker 的配置文件部署给所有用户。

恭喜，您已成功完成 Docker 的管理员实施流程。

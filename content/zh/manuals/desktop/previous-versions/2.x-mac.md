## Docker Desktop Community 2.5.0.0
2020-11-02

Docker Desktop 2.5.0.0 包含 Kubernetes 升级。安装此版本后，您的本地 Kubernetes 集群将被重置。

### 新功能

- 拥有付费 Docker 订阅的用户现在可以在 Docker Desktop 的远程仓库选项卡中查看漏洞扫描报告。
- Docker Desktop 为拥有付费 Docker 订阅的用户引入了支持选项。

### 安全

- 修复了因证书检查不充分导致的本地权限提升漏洞。参见 [CVE-2021-3162](https://cve.mitre.org/cgi-bin/cvename.cgi?name=CVE-2021-3162)。

### 升级

- [Linux kernel 5.4.39](https://hub.docker.com/layers/linuxkit/kernel/5.4.39-f39f83d0d475b274938c86eaa796022bfc7063d2/images/sha256-8614670219aca0bb276d4749e479591b60cd348abc770ac9ecd09ee4c1575405?context=explore)
- [Docker Compose CLI 1.0.1](https://github.com/docker/compose-cli/releases/tag/v1.0.1)
- [Snyk v1.421.1](https://github.com/snyk/snyk/releases/tag/v1.421.1)
- [Go 1.15.2](https://github.com/golang/go/releases/tag/go1.15.2)
- [Kubernetes 1.19.3](https://github.com/kubernetes/kubernetes/releases/tag/v1.19.3)

### 错误修复和小改进

- 将"Run Diagnostics"重命名为"Get support"。
- 移除了 BlueStacks 警告消息。修复 [docker/for-mac#4863](https://github.com/docker/for-mac/issues/4863)。
- 在共享卷包含大量文件的情况下，加快了容器启动速度。修复 [docker/for-mac#4957](https://github.com/docker/for-mac/issues/4957)。
- 文件共享：修复了只读文件的所有权更改问题。修复 [docker/for-mac#4989](https://github.com/docker/for-mac/issues/4989)、[docker/for-mac#4964](https://github.com/docker/for-mac/issues/4964)。
- 文件共享：生成 `ATTRIB` inotify 事件以及 `MODIFY` 事件。修复 [docker/for-mac#4962](https://github.com/docker/for-mac/issues/4962)。
- 文件共享：对于不支持的模式，`fallocate` 返回 `EOPNOTSUPP`。修复 `minio`。修复 [docker/for-mac#4964](https://github.com/docker/for-mac/issues/4964)。
- 文件共享：修复了可能过早关闭文件句柄的问题。
- 当与容器共享 Linux 目录（`/var`、`/bin` 等）时，Docker Desktop 避免监视主机文件系统中的路径。
- 当将文件共享到容器中时（例如 `docker run -v ~/.gitconfig`），Docker Desktop 不监视父目录。修复 [docker/for-mac#4981](https://github.com/docker/for-mac/issues/4981)、[docker/for-mac#4975](https://github.com/docker/for-mac/issues/4975)。
- 修复了与 NFS 挂载相关的问题。修复 [docker/for-mac#4958](https://github.com/docker/for-mac/issues/4958)。
- 允许符号链接指向共享卷之外。修复 [docker/for-mac#4862](https://github.com/docker/for-mac/issues/4862)。
- 诊断：避免在 Kubernetes 处于损坏状态时挂起。
- Docker Desktop 现在在共享文件系统上的 `chmod(2)` 调用中支持 `S_ISUID`、`S_ISGID` 和 `S_ISVTX`。修复 [docker/for-mac#4943](https://github.com/docker/for-mac/issues/4943)。

## Docker Desktop Community 2.4.0.0
2020-09-30

Docker Desktop 2.4.0.0 包含 Kubernetes 升级。安装此版本后，您的本地 Kubernetes 集群将被重置。

### 新功能

- [Docker Compose CLI - 0.1.18](https://github.com/docker/compose-cli)，支持通过 ECS 和 ACI 在云端使用 Compose 和卷。
- Docker 在 Docker Dashboard 中引入了新的镜像视图。镜像视图允许用户查看 Hub 镜像、拉取镜像并管理磁盘上的本地镜像，包括清理不需要和未使用的镜像。要访问新的镜像视图，从 Docker 菜单中选择 **Dashboard** > **Images**。
- Docker Desktop 现在在恢复出厂设置后默认启用 BuildKit。要恢复到旧的 `docker build` 体验，请转到 **Preferences** > **Docker Engine**，然后禁用 BuildKit 功能。
- [Amazon ECR Credential Helper](https://github.com/awslabs/amazon-ecr-credential-helper/releases/tag/v0.4.0)
- Docker Desktop 现在在主机上有大量文件事件以及运行 Kubernetes 时使用更少的 CPU，参见 [docker/roadmap#12](https://github.com/docker/roadmap/issues/12)。
- Docker Desktop 现在默认使用 gRPC-FUSE 进行文件共享。这比 osxfs 使用更少的 CPU，特别是当主机上有大量文件事件时。要切换回 `osxfs`，请转到 **Preferences** > **General** 并禁用 gRPC-FUSE。

### 升级

- [Docker 19.03.13](https://github.com/docker/docker-ce/releases/tag/v19.03.13)
- [Docker Compose 1.27.4](https://github.com/docker/compose/releases/tag/1.27.4)
- [Go 1.14.7](https://github.com/golang/go/releases/tag/go1.14.7)
- [Alpine 3.12](https://alpinelinux.org/posts/Alpine-3.12.0-released.html)
- [Kubernetes 1.18.8](https://github.com/kubernetes/kubernetes/releases/tag/v1.18.8)
- [Qemu 4.2.0](https://git.qemu.org/?p=qemu.git;a=tag;h=1e4aa2dad329852aa6c3f59cefd65c2c2ef2062c)

### 错误修复和小改进

- macOS 10.13 上的 Docker Desktop 现已弃用。
- 移除了旧版 Kubernetes 上下文 `docker-for-desktop`。应改用上下文 `docker-desktop`。修复 [docker/for-win#5089](https://github.com/docker/for-win/issues/5089) 和 [docker/for-mac#4089](https://github.com/docker/for-mac/issues/5089)。
- 将应用程序添加到 dock 并单击它，如果 Docker 已在运行，将启动容器视图。
- 添加了通过 Qemu 4.2.0 模拟 Risc-V 的支持。
- 移除了 `10240` 的文件描述符限制（`setrlimit`）。我们现在依赖内核通过 `kern.maxfiles` 和 `kern.maxfilesperproc` 施加限制。
- 修复了通过从 `hyperkit` 移除串行控制台导致的 Mac CPU 使用率问题，参见 [docker/roadmap#12]( https://github.com/docker/roadmap/issues/12#issuecomment-663163280)。要在虚拟机中打开 shell，请使用 `nc -U ~/Library/Containers/com.docker.docker/Data/debug-shell.sock`。
- 复制容器日志到剪贴板时不包含 ansi 颜色代码。修复 [docker/for-mac#4786](https://github.com/docker/for-mac/issues/4786)。
- 修复了登录时自动启动的问题。参见 [docker/for-mac#4877] 和 [docker/for-mac#4890]。
- 修复了当用户名过长时应用程序无法启动的错误。
- 修复了将 `/usr` 等目录添加到文件共享列表会阻止 Desktop 启动的错误。修复 [docker/for-mac#4488](https://github.com/docker/for-mac/issues/4488)
- 修复了在 Docker `daemon.json` 中指定 `hosts` 时应用程序启动的问题。参见 [docker/for-win#6895](https://github.com/docker/for-win/issues/6895#issuecomment-637429117)
- Docker Desktop 在容器启动时始终同步刷新文件系统缓存。参见 [docker/for-mac#4943](https://github.com/docker/for-mac/issues/4943)。
- Compose-on-Kubernetes 不再包含在 Docker Desktop 安装程序中。您可以从 compose-on-kubernetes [发布页面](https://github.com/docker/compose-on-kubernetes/releases) 单独下载。

### 已知问题

-  在使用 `docker-compose` 与命名卷和 gRPC FUSE 时存在已知问题：第二次及后续调用 `docker-compose up` 将因卷路径带有 `/host_mnt` 前缀而失败。要解决此问题，请在设置中切换回 `osxfs`。参见 [docker/for-mac#4859](https://github.com/docker/for-mac/issues/4859)。
- 启用 Kubernetes 时存在已知问题，设置 UI 有时无法更新 Kubernetes 状态。要解决此问题，请关闭并重新打开窗口。
- 切换用户时存在罕见的已知问题，镜像视图继续显示前一个用户的仓库。要解决此问题，请关闭并重新打开窗口。

## Docker Desktop Community 2.2.0.0
2020-01-21

Docker Desktop 2.2.0.0 包含 Kubernetes 升级。安装此版本后，您的本地 Kubernetes 集群将被重置。

### 升级

- [Docker Compose 1.25.2](https://github.com/docker/compose/releases/tag/1.25.2)
- [Kubernetes 1.15.5](https://github.com/kubernetes/kubernetes/releases/tag/v1.15.5)
- Linux kernel 4.19.76
- [QEMU 4.0.1](https://github.com/docker/binfmt)

### 新功能

- **Docker Desktop Dashboard：** 新的 Docker Desktop **Dashboard** 提供用户友好的界面，使您能够与容器和应用程序交互，并直接从 UI 管理应用程序的生命周期。此外，它允许您访问日志、查看容器详情并监控资源利用率以探索容器行为。
  有关新 Dashboard UI 的详细信息，请参见 [Docker Desktop Dashboard](../_index.md)。

- 为 Docker Desktop **Preferences** 菜单引入了新的用户界面。
- 重启、重置和卸载选项现在在 **Troubleshoot** 菜单上可用。
- 添加了在 Docker Desktop **Dashboard** UI 中启动和停止现有基于 Compose 的应用程序并查看组合日志的功能。

### 错误修复和小改进

- 为 Docker Compose 添加了缺失的 `fish` shell 补全。修复 [docker/for-mac#3795](https://github.com/docker/for-mac/issues/3795)。
- 修复了不允许用户在 **Preferences** > **Daemon** 窗口中复制和粘贴文本的错误。修复 [docker/for-mac#3798](https://github.com/docker/for-mac/issues/3798)。
- 在 Docker API 代理中添加了对 `Expect: 100-continue` 头的支持。某些 HTTP 客户端（如 `curl`）在负载较大时会发送此头，例如在创建容器时。修复 [moby/moby#39693](https://github.com/moby/moby/issues/39693)。
- 向 **Settings** 和 **Troubleshoot** 窗口添加了加载覆盖层以防止编辑冲突。
- 当 Kubernetes 未激活时，禁用 **Reset Kubernetes** 按钮。
- 改进了 **Settings** 和 **Troubleshoot** UI 中的导航。
- 修复了 UEFI 引导菜单中有时导致 Docker Desktop 在重启期间挂起的错误。修复 [docker/for-mac#2655](https://github.com/docker/for-mac/issues/2655) 和 [docker/for-mac#3921](https://github.com/docker/for-mac/issues/3921)。
- Docker Desktop 现在允许用户在容器内访问主机的 SSH 代理。修复 [docker/for-mac#410](https://github.com/docker/for-mac/issues/410)
- Docker Machine 不再包含在 Docker Desktop 安装程序中。您可以从 [Docker Machine releases](https://github.com/docker/machine/releases) 页面单独下载。
- 修复了在 macOS Catalina 上运行较旧硬件的虚拟机启动时出现 `processor does not support desired secondary processor-based controls` 错误而失败的问题。
- 修复了容器使用 `overlay` 网络时的端口转发问题。
- 修复了当容器有多个端口且外部端口号任意或尚未配置时容器启动错误。例如，`docker run -p 80 -p 443 nginx`。修复 [docker/for-win#4935](https://github.com/docker/for-win/issues/4935) 和 [docker/compose#6998](https://github.com/docker/compose/issues/6998)。
- 修复了共享重叠目录时发生的问题。
- 修复了阻止用户更改虚拟机磁盘镜像位置的错误。
- Docker Desktop 不再在目录上注入 `inotify` 事件，因为这些可能导致容器内的挂载点消失。修复 [docker/for-mac#3976](https://github.com/docker/for-mac/issues/3976)。
- 修复了当 Kubernetes 配置文件不完整时导致 Docker Desktop 启动失败的问题。
- 修复了尝试通过 Docker Desktop 登录 Docker 时有时因 `Incorrect authentication credentials` 错误而失败的问题。修复 [docker/for-mac#4010](https://github.com/docker/for-mac/issues/4010)。

### 已知问题

- 当您启动 Docker Compose 应用程序，然后启动与 Compose 应用程序同名的 Docker App 时，Docker Desktop 在 Dashboard 上只显示一个应用程序。但是，当您展开应用程序时，属于两个应用程序的容器都会显示在 Dashboard 上。

- 当您在 Kubernetes 上部署具有多个容器的 Docker App 时，Docker Desktop 将每个 Pod 显示为 Dashboard 上的应用程序。

### 新功能

* 在 Daemon **Preferences** 菜单中选择"Experimental features"复选框将为 Docker 守护进程和 Docker CLI 开启实验性功能。
* 改进了 `com.docker.osxfs trace` 性能分析命令的可靠性。用户现在可以运行 `com.docker.osxfs trace --summary` 选项来获取操作的高级摘要，而不是接收所有操作的跟踪。
* Docker Desktop 现在在 Mac 上支持大型 DNS 资源记录列表。修复 [docker/for-mac#2160](https://github.com/docker/for-mac/issues/2160#issuecomment-431571031)。

### 实验性功能

> 实验性功能提供了对未来产品功能的早期访问。这些功能仅用于测试和反馈，因为它们可能在版本之间更改而不发出警告，或者可能在未来版本中完全删除。实验性功能不得在生产环境中使用。Docker 不为实验性功能提供支持。

Docker Desktop Community 2.1.0.0 包含以下实验性功能。

* Docker App：Docker App 是一个 CLI 插件，帮助配置、共享和安装应用程序。
* Docker Buildx：Docker Buildx 是一个用于扩展 BuildKit 构建功能的 CLI 插件。有关更多信息，请参见 [Build 页面](/manuals/build/_index.md)。

### 错误修复和小改进

* Docker Desktop 现在允许用户公开特权 UDP 端口。[docker/for-mac#3775](https://github.com/docker/for-mac/issues/3775)
* 修复了在不使用凭据助手时运行某些 Docker 命令可能失败的问题。[docker/for-mac#3785](https://github.com/docker/for-mac/issues/3785)
* 更改了主机的 kubernetes 上下文，使 `docker run -v .kube:kube ... kubectl` 可以工作。
* 将本地 Kubernetes 集群上的 `cluster-admin` 角色限制在 `kube-system` 命名空间。
* 减少了虚拟机启动时间。不再每次虚拟机启动时创建 swap。
* 修复了使用 VPNkit 子网的 Kubernetes 安装。
* 修复了在 Windows 上收集诊断信息时进程输出未重定向到 stdout 的问题，这有时会导致崩溃。
* 向虚拟机添加了 `/etc/machine-id`。修复 [docker/for-mac#3554](https://github.com/docker/for-mac/issues/3554)。
* Docker Desktop 不再每 10 秒发送一次 `docker-desktop.<domain>` 的 DNS 查询。它现在依赖主机的 DNS 域搜索顺序，而不是尝试在虚拟机内复制它。
* 移除了使用电子邮件地址作为用户名登录的功能，因为 Docker 命令行不支持此功能。
* Docker Desktop 现在允许在容器内运行 Docker registry。修复 [docker/for-mac#3611](https://github.com/docker/for-mac/issues/3611)。
* 修复了 DNS 解析器的稳定性问题。
* Docker Desktop 截断超过 512 字节的 UDP DNS 响应。
* 修复了启动 Kubernetes 时使用的 localhost 端口 8080。修复 [docker/for-mac#3522](https://github.com/docker/for-mac/issues/3522)。
* 改进了错误消息传递：Docker Desktop 在不适当时不再提示用户运行诊断或恢复出厂设置。
* Kubernetes：Docker Desktop 现在使用 kubelet 的默认最大 pod 数。[docker/for-mac#3453](https://github.com/docker/for-mac/issues/3453)。
* 修复了 DockerHelper 崩溃。[docker/for-mac#3470](https://github.com/docker/for-mac/issues/3470)。
* 修复了指定 IP 的特权端口绑定。[docker/for-mac#3464](https://github.com/docker/for-mac/issues/3464)
* 修复了诊断中的服务日志收集。
* Docker Desktop 现在收集 `/etc/hosts` 以帮助诊断。
* 当两个服务有共同的公开端口时，Docker Desktop 现在为第二个服务公开可用端口。[docker/for-mac#3438](https://github.com/docker/for-mac/issues/3438)。
* Docker Desktop 确保 localhost 解析为 127.0.0.1。这与 [docker/for-mac#2990](https://github.com/docker/for-mac/issues/2990#issuecomment-443097942)、[docker/for-mac#3383](https://github.com/docker/for-mac/issues/3383) 相关。

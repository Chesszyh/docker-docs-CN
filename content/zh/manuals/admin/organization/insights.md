“洞察”（Insights）可帮助管理员可视化并了解 Docker 在其组织内的使用情况。通过“洞察”，管理员可以确保其团队具备充分利用 Docker 潜力的能力，从而提高整个组织的生产力和效率。

主要优势包括：

- 统一的工作环境。在各团队间建立并维护标准化的配置。
- 最佳实践。推广并强制执行使用指南以确保最佳性能。
- 提高可见性。监控并推动组织配置和策略的采用。
- 优化许可证使用。确保开发人员能够访问 Docker 订阅提供的先进功能。

## 前提条件

- [Docker Business 订阅](../../subscription/details.md#docker-business)
- 管理员必须为用户 [强制执行登录](/security/for-admins/enforce-sign-in/)
- 由您的客户成功经理启用“洞察”功能

## 查看组织用户的洞察数据

要访问“洞察”，您必须联系您的客户成功经理以启用该功能。功能启用后，请按照以下步骤访问“洞察”：

1. 登录 [Docker Home](https://app.docker.com/) 并选择您的组织。
2. 选择 **Insights**（洞察），然后选择数据的时间段。

> [!NOTE]
>
> “洞察”数据不是实时的，而是每天更新。在“洞察”页面的右上角，可以查看 **Last updated**（上次更新）日期，以了解数据的最后更新时间。

您可以在以下图表中查看数据：

 - [Docker Desktop 用户](#docker-desktop-用户)
 - [构建](#构建)
 - [容器](#容器)
 - [Docker Desktop 使用情况](#docker-desktop-使用情况)
 - [Docker Hub 镜像](#docker-hub-镜像)
 - [扩展](#扩展)

## 导出 Docker Desktop 用户数据

您可以将 Docker Desktop 用户数据导出为 CSV 文件：

1. 打开 [Docker Home](https://app.docker.com) 并在 **Choose profile**（选择个人资料）页面选择您的组织。
2. 在左侧导航菜单中选择 **Admin Console**（管理控制台）。
3. 选择 **Desktop insights**（桌面洞察）。
4. 为您的洞察数据选择时间范围：**1 Week**（1 周）、**1 Month**（1 个月）或 **3 Months**（3 个月）。
5. 选择 **Export**（导出），然后从下拉菜单中选择 **Docker Desktop users**（Docker Desktop 用户）。

导出文件将自动下载。打开文件即可查看导出数据。

### 了解导出数据

Docker Desktop 用户导出文件包含以下数据点：

- Name：用户姓名
- Username：用户的 Docker ID
- Email：与用户 Docker ID 关联的电子邮件地址
- Type：用户类型
- Role：用户 [角色](/manuals/security/for-admins/roles-and-permissions.md)
- Teams：用户所属组织内的团队
- Date Joined：用户加入组织的日期
- Last Logged-In Date：用户上次使用 Web 浏览器登录 Docker 的日期（包括 Docker Hub 和 Docker Home）
- Docker Desktop Version：用户安装的 Docker Desktop 版本
- Last Seen Date：用户上次使用 Docker Desktop 应用程序的日期
- Opted Out Analytics：用户是否在 Docker Desktop 中选择了退出 [发送使用统计数据](/manuals/security/for-admins/hardened-desktop/settings-management/settings-reference.md#send-usage-statistics) 设置

## 洞察数据故障排除

如果您在“洞察”数据方面遇到问题，请考虑以下解决方案来解决常见问题。

- 将用户更新到最新版本的 Docker Desktop。

   使用 Docker Desktop 4.16 或更低版本的用户将不会显示数据。此外，旧版本可能无法提供所有数据。请确保所有用户都安装了最新版本的 Docker Desktop。

- 为所有用户在 Docker Desktop 中启用 **Send usage statistics**（发送使用统计数据）。

   如果用户选择了退出发送 Docker Desktop 的使用统计数据，那么他们的使用数据将不会包含在“洞察”中。要为所有用户大规模管理该设置，您可以使用 [设置管理](/desktop/hardened-desktop/settings-management/) 并启用 `analyticsEnabled` 设置。

- 确保用户使用的是 Docker Desktop，而不是独立版本的 Docker Engine。

   只有 Docker Desktop 能为“洞察”提供数据。如果用户在 Docker Desktop 之外安装了 Docker Engine，Docker Engine 将不会为该用户提供数据。

- 确保用户登录的是与您组织关联的账户。

   未登录与您组织关联的账户的用户不会体现在数据中。要确保用户使用与您组织关联的账户登录，您可以 [强制执行登录](/security/for-admins/enforce-sign-in/)。

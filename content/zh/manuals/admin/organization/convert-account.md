您可以将现有的用户账户转换为组织。如果您需要多个用户访问您的账户及其关联的存储库，这将非常有用。将其转换为组织后，您可以通过 [团队](manage-a-team.md) 和 [角色](roles-and-permissions.md) 更好地控制这些用户的权限。

当您将用户账户转换为组织时，该账户将迁移到 Docker Team 订阅。

> [!IMPORTANT]
>
> 一旦将您的账户转换为组织，就无法再将其恢复为用户账户。

## 前提条件

在将用户账户转换为组织之前，请确保满足以下要求：

- 您要转换的用户账户不能是公司、任何团队或组织的成员。您必须将该账户从所有团队、组织或公司中移除。

    步骤如下：
    1. 导航到 **My Hub**，然后选择您需要退出的组织。
    2. 在 **Members**（成员）选项卡中找到您的用户名。
    3. 选择 **More options**（更多选项）菜单，然后选择 **Leave organization**（退出组织）。

    如果该用户账户是任何组织或公司的唯一所有者，请为另一名用户分配所有者角色，然后将您自己从组织或公司中移除。

- 您必须准备好一个单独的 Docker ID，以便在转换期间将其分配为该组织的所有者。

    如果您想将您的用户账户转换为组织账户且没有其他用户账户，则需要创建一个新的用户账户并将其指定为新组织的所有者。被分配所有者角色后，该用户账户将拥有配置和管理组织的完整管理访问权限。转换后，您可以为更多用户分配所有者角色。

## 将账户转换为组织的影响

请考虑转换账户的以下影响：

- 此过程会移除该账户的电子邮件地址，组织所有者将改为接收通知邮件。转换后，您将能够将移除的电子邮件地址用于另一个账户。

- 当前订阅将取消，您的新订阅将开始。

- 存储库命名空间和名称不会更改，但转换账户会移除所有存储库协作者。转换账户后，您需要将这些用户添加为团队成员。

- 现有的自动构建将显示为是由添加到组织的第一个所有者设置的。有关添加第一个所有者的步骤，请参阅 [将账户转换为组织](#convert-an-account-into-an-organization)。

- 您作为第一个所有者添加的用户账户将拥有配置和管理组织的完整管理访问权限。

- 要将用户的个人访问令牌 (PAT) 转移到您转换后的组织，您必须将该用户指定为组织所有者。这将确保与该用户账户关联的任何 PAT 都会转移到组织所有者。

> [!TIP]
>
> 为了避免在转换账户或更改所有权时可能中断个人访问令牌的服务，建议使用 [组织访问令牌](/manuals/security/for-admins/access-tokens.md)。组织访问令牌与组织关联，而非单个用户账户。

## 将账户转换为组织

在将账户转换为组织之前，请确保您已经：

- 已将您的用户账户从任何公司、团队或组织中移除
- 在转换账户之前已创建了一个新的 Docker ID

详情请参阅 [前提条件](#prerequisites) 部分。

1. 登录 [Docker Home](https://app.docker.com/login)。
2. 选择右上角的头像，然后选择 **Account settings**（账户设置）。
3. 在 **Settings**（设置）部分，选择 **Convert**（转换）。
4. 查看显示的关于转换用户账户的警告。此操作无法撤销，且会对您的资产和账户产生重大影响。
5. 输入 **Username of new owner**（新所有者的用户名）以设置组织所有者。这是管理该组织的业务账户，也是转换后访问组织设置的唯一途径。您不能使用与要转换的账户相同的 Docker ID。
6. 选择 **Confirm**（确认）。新所有者将收到一封通知邮件。使用该所有者账户登录并管理新组织。

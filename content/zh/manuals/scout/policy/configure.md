某些策略类型是可配置的。这意味着您可以创建具有自己配置参数的该策略类型的新自定义版本。如果您需要暂时忽略某项策略，也可以将其禁用；如果某项策略不符合您的需求，则可以将其彻底删除。

> [!NOTE]
> 如果您删除或自定义了某项策略，则默认策略配置的历史评估结果将被删除。

## 添加策略

要添加新策略，请选择您想要自定义的策略类型。所有自定义策略都以一种策略类型作为基础。

您可以编辑新策略的显示名称和描述，以帮助更好地传达策略的合规和不合规状态。您无法更改策略类型的名称，只能更改其显示名称。

策略的可选配置参数取决于您正在编辑的策略类型。有关更多信息，请参阅 [策略类型](/manuals/scout/policy/_index.md#policy-types)。

添加策略：

1. 转到 Docker Scout 控制面板中的 [Policies 页面](https://scout.docker.com/reports/policy)。
2. 选择 **Add policy** 按钮以打开策略配置屏幕。
3. 在策略配置屏幕上，找到您要配置的策略类型，然后选择 **Configure** 以打开策略配置页面。

   - 如果 **Configure** 按钮为灰色，则表示当前策略没有可配置参数。
   - 如果按钮显示为 **Integrate**，则表示在启用该策略之前需要进行设置。选择 **Integrate** 将引导您进入集成的设置指南。

4. 更新策略参数。
5. 保存更改：

   - 选择 **Save policy** 以提交更改并为当前组织启用该策略。
   - 选择 **Save and disable** 以保存策略配置而不启用它。

## 编辑策略

编辑策略允许您修改其配置，而无需从头开始创建新策略。当由于需求演变或组织合规目标发生变化而需要调整策略参数时，这非常有用。

要编辑策略：

1. 转到 Docker Scout 控制面板中的 [Policies 页面](https://scout.docker.com/reports/policy)。
2. 选择您要编辑的策略。
3. 选择 **Edit** 按钮。
4. 更新策略参数。
5. 保存更改。

## 禁用策略

禁用策略后，该策略的评估结果将被隐藏，不再出现在 Docker Scout 控制面板或 CLI 中。禁用策略不会删除历史评估结果，因此如果您改变主意并在以后重新启用策略，早期的评估结果仍将可用。

要禁用策略：

1. 转到 Docker Scout 控制面板中的 [Policies 页面](https://scout.docker.com/reports/policy)。
2. 选择您要禁用的策略。
3. 选择 **Disable** 按钮。

## 删除策略

删除策略后，该策略的评估结果也将被删除，并且不再出现在 Docker Scout 控制面板或 CLI 中。

要删除策略：

1. 转到 Docker Scout 控制面板中的 [Policies 页面](https://scout.docker.com/reports/policy)。
2. 选择您要删除的策略。
3. 选择 **Delete** 按钮。

## 恢复已删除的策略

如果您删除了某项策略，可以按照 [添加策略](#add-a-policy) 中的步骤重新创建它。在策略配置屏幕上，在您希望重新创建的已删除策略上选择 **Configure**。

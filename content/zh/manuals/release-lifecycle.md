本页面详细介绍了 Docker 的产品发布生命周期以及 Docker 如何定义每个阶段。它还提供了有关产品退役流程的信息。功能和产品可能会经历这些阶段中的部分或全部。

## 生命周期阶段

| 生命周期阶段 | 客户可用性 | 支持可用性 | 限制 | 退役 |
| --- | --- | ---- | ---| ---|
|[Experimental（实验性）](#experimental)| 有限的可用性 | 社区支持 |软件可能存在限制、Bug 和/或稳定性问题 | 可能会在不另行通知的情况下停止 |
|[Beta](#beta) | 全部或参与 Beta 反馈计划的用户 | 社区支持 | 软件可能存在限制、Bug 和/或稳定性问题 | 可能会在不另行通知的情况下停止 |
| [Early Access (EA)（早期访问）](#early-access-ea) | 全部或参与早期访问反馈计划的用户 | 全面 | 软件可能存在限制、Bug 和/或稳定性问题。这些限制将被记录。 | 遵循 [退役流程](#retirement-process) |
| [General Availability (GA)（正式发布）](#general-availability-ga) | 全部 | 全面 | 对于受支持的用例，几乎没有或没有限制 | 遵循 [退役流程](#retirement-process) |

### Experimental（实验性）

Experimental 产品是 Docker 目前正在试验的功能。访问实验性功能的客户有机会测试、验证未来功能并提供反馈。这有助于我们将精力集中在能为客户提供最大价值的地方。

**客户可用性：** 实验性功能的可用性是有限的。部分用户可能无法访问、可以访问一个或多个实验性功能。

**支持：** 对实验性功能的支持是通过社区支持渠道和论坛尽力提供的。

**限制：** 实验性功能可能存在潜在的重大限制，例如功能限制、性能限制和 API 限制。功能和编程接口可能会随时更改，恕不另行通知。

**退役：** 在实验期间，Docker 将决定是否在其生命周期内继续提供该产品。我们保留在不另行通知的情况下随时更改实验性产品或功能的范围或停止该产品或功能的权利，如我们的 [订阅服务协议](https://www.docker.com/legal/docker-subscription-service-agreement) 中所述。

### Beta

Beta 产品是潜在未来产品或功能的初始版本。参与我们 Beta 计划的客户有机会测试、验证未来功能并提供反馈。这有助于我们将精力集中在能为客户提供最大价值的地方。

**客户可用性：** 参与 Beta 发布是通过邀请或通过使用产品中明确标识的 Beta 功能进行的。Beta 邀请可以是公开的，也可以是私有的。

**支持：** 对 Beta 功能的支持是通过社区支持渠道和论坛尽力提供的。

**限制：** Beta 版本可能存在潜在的重大限制，例如功能限制、性能限制和 API 限制。功能和编程接口可能会随时更改，恕不另行通知。

**退役：** 在 Beta 期间，Docker 将决定是否在其生命周期内继续提供该产品。我们保留在不另行通知的情况下随时更改 Beta 产品或功能的范围或停止该产品或功能的权利，如我们的 [订阅服务协议](https://www.docker.com/legal/docker-subscription-service-agreement) 中所述。

### Early Access (EA)（早期访问）

Early Access 产品是可能具有潜在功能限制的产品或功能，作为增量推出策略的一部分，对特定用户组启用。它们已准备好发布给全世界，尚待微调。

**客户可用性：** Early Access 功能可以向所有客户或特定用户群推出，作为现有功能的补充或替代。

**支持：** Early Access 产品提供与正式发布（General Availability）功能和产品相同级别的支持。

**限制：** Early Access 版本可能存在潜在的重大限制，例如功能限制、性能限制和 API 限制，尽管这些限制将被记录。功能和编程接口的重大变更将遵循下文概述的 [退役流程](#retirement-process)。

**退役：** 如果我们在正式发布之前退役 Early Access 产品，我们将努力遵循下文概述的 [退役流程](#retirement-process)。

### General Availability (GA)（正式发布）

General Availability 产品是功能齐全的产品或功能，对所有 Docker 客户公开开放。

**客户可用性：** 所有 Docker 用户都可以根据其订阅级别访问 GA 产品。

**限制：** General Availability 功能和产品对于受支持的用例几乎没有或没有限制。

**支持：** 所有 GA 产品均受全面支持，如我们的 [支持页面](https://www.docker.com/support/) 所述。

**退役：** General Availability 产品遵循下文概述的 [退役流程](#retirement-process)。


## 退役流程

退役或弃用功能的决定遵循严格的流程，包括了解需求、使用情况、功能退役的影响，最重要的是客户反馈。我们的目标是将资源投入到能为大多数客户增加最大价值的领域。

Docker 致力于在与客户互动时保持清晰、透明和主动，尤其是在涉及平台变更时。为此，我们在退役功能时将尽最大努力遵循以下准则：

- **提前通知：** 对于主要功能或产品的退役，我们将尝试至少提前 6 个月通知客户。
- **可行的替代方案：** Docker 将在退役功能时努力为客户提供可行的替代方案。这可能是来自 Docker 的替代产品或来自第三方提供商的推荐替代方案。在可能且适当的情况下，Docker 将自动将客户迁移到已退役功能的替代方案。
- **持续支持：** Docker 承诺在功能退役日期之前提供持续支持。

在特殊情况下，例如为了保护我们平台的完整性或客户及其他人的安全而必须进行的基本更改，我们可能需要加快功能退役的时间表。在这些情况下，尽快进行这些更改非常重要。

同样，集成的第三方软件或服务可能由于第三方决定更改或退役其解决方案而需要退役。在这些情况下，退役的步伐将超出我们的控制范围。

然而，即使在这些情况下，我们将尽可能提供提前通知。

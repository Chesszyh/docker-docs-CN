> [!IMPORTANT]
>
> 自 2024 年 12 月 10 日起，增强服务账户附加组件不再提供。现有的服务账户协议将在其当前期限到期前继续有效，但增强服务账户附加组件的新购买或续订不再可用，客户必须在新订阅下续订。
>
> Docker 建议过渡到[组织访问令牌（OATs）](../security/for-admins/access-tokens.md)，它可以提供类似的功能。

服务账户是用于自动管理容器镜像或容器化应用程序的 Docker ID。服务账户通常用于自动化工作流程，不与组织中的成员共享 Docker ID。服务账户的常见用例包括镜像 Docker Hub 上的内容，或将镜像拉取与您的 CI/CD 流程绑定。

## 增强服务账户附加组件层级

有关增强服务账户附加组件的详细信息，请参阅下表：

| 层级 | 每日拉取速率* |
| ------ | ------ |
| 1 | 5,000-10,000 |
| 2 | 10,000-25,000 |
| 3 | 25,000-50,000 |
| 4 | 50,000-100,000 |
| 5 | 100,000+ |

<sub>*服务账户在一年内最多 20 天内可超出拉取量最多 25%，而不会产生额外费用。可根据请求提供消费报告。<sub>
